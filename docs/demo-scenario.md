## 1. 前提

環境構築・起動手順は本ファイルでは繰り返しません。リポジトリ直下 [`CLAUDE.md`](../CLAUDE.md) の「セットアップ手順」に従い、バックエンド（`uvicorn app.main:app --reload`）・フロントエンド（`npm run dev`）の両方を起動してください。既定設定（`DB_BACKEND=sqlite`, `USE_KRE_STUB=true`, `AUTH_MODE=mock`）のままで、外部サービスに接続せずに本手順を再現できます。

> [!note] ④作戦シートと AI 生成について
> 本手順が参照する代表案件の④作戦シートは、シード投入済みの保存データを表示するため、**既定設定のまま（Azure OpenAI 未設定でも）確認まで完走できます**。作戦シート画面の [🔁再生成]／[AIで交渉ポイントを生成する] ボタンで **AI を実際に呼び出す操作だけ** は Azure OpenAI（`AZURE_OPENAI_*`）の設定が必要で、未設定のまま実行すると 502 になります（この AI 再生成はデモの完走には不要です）。

シードデータは `python -m app.ingest.seed`（`backend/app/ingest/seed.py`）で投入します。本手順書が参照する代表案件は、このシード投入だけで揃います（追加の手入力は不要）。

## 2. 代表案件サマリ（①②）

issue #8 の要件「代表案件を1〜2件定義」に対応する2系統の代表案件です。

### 代表案件① No.123457-a — 相場なし・過去実績なしの新規案件

- 取引先: 東日本ミート（`supplier_id=2`、国産鶏・豚が主力。決裁が遅い〔2週間〕という交渉メモ付き）
- 商材: 国産鶏むね肉（チルド、`spec_id=4`）
- ステータス: 交渉中／値上げ要請（現行¥670 → 提案¥690）
- 特徴: この商材（`infomart_code=113803`）は農水省相場CSV（`market_rates_sample.csv`）に収録がなく、かつ東日本ミートとの過去交渉実績も本シードには存在しません。**相場情報が「未登録」のまま、担当者が手入力で埋めて進める**という、最も基本的な新規案件の流れを見せるのに適しています。

### 代表案件② No.123458-a → No.123459-a — 申し送り再利用ペア

- 取引先: グローバルフーズ商事（`supplier_id=3`、輸入牛肉業者。相見積で丸紅畜産を動かせる／小口NG という交渉メモ付き）
- 商材: 輸入牛かたばら（豪州産・チルド、`spec_id=5`、`infomart_code=112503`。グローバルフーズ商事の輸入牛肉ライン）
- **No.123458-a**（2026Q1・完了）: 決着単価¥1,215、結果記録済み（`negotiation_results.result_id=5`）。
  - 所感（`staff_memo`）: 「輸入牛が為替高・供給ひっ迫で強気。相見積カードで¥1215に収めた」
  - 申し送り（`handover_note`）: 「次回は別ソースの相見積を早めに用意。小口NGなので数量をまとめて臨む」
- **No.123459-a**（2026Q3・交渉中）: 同一取引先×同一商材の次回案件。現行¥1,215 → 提案¥1,260。保存済みの作戦シート（`strategy_sheets.sheet_id=7`）をシード同梱しており、AI 生成を呼ばずに④画面を確認できます。
- 特徴: この商材も相場CSVには収録がなく相場は「未登録」ですが、**過去実績（No.123458-a）が1件あるため、②情報収集画面の「過去経緯」パネルと④作戦シートに前回の申し送りが自動反映される**流れを見せられます（判断継承ループ BR-10 の実演）。

## 4. 一連フローの辿り方手順

画面遷移: `/cases`（①案件一覧）→ `/cases/{caseNo}`（案件サマリ）→ `/cases/{caseNo}/collect`（②情報収集）→ `/cases/{caseNo}/lines`（③3ライン算出）→ `/cases/{caseNo}/strategy`（④作戦シート）→ `/cases/{caseNo}/result`（⑤結果記録）。ステップは `[caseNo]/layout.tsx` の StepIndicator が常時表示します。

以下、**代表案件① No.123457-a** を例に手順を示します（②③④⑤は「案件ワークスペース」内で連続して遷移します）。

### ① 案件一覧（`/cases`）

- 案件一覧に No.123457-a（東日本ミート・国産鶏むね肉・交渉中）が表示されることを確認します。
- **#4 取引先選択（issue #4）の確認**: 「新規案件作成」モーダル（`CreateCaseModal.tsx`）を開き、「取引先企業」欄が自由入力ではなく候補リストからの選択式（combobox）になっていることを確認します。`api.listSuppliers()` でマスタ（東日本ミート含む4社）から選びます。

### ② 情報収集（`/cases/{caseNo}/collect`）

相場・過去経緯・自社計画の3カラムを1画面で確認します。

- **#3・#7 相場未登録の表示（issue #3・#7）の確認**: No.123457-a の「相場情報」パネルで、価格が0円ではなく **「未登録」バッジ**（amber色）と「相場データがまだ登録されていません。手入力で登録してください。」の案内が出ることを確認します（`RateInfo.registered` による判定。価格0円と未登録は区別されます）。
- **#2 相場手入力（issue #2）の確認**: 同パネルの「手入力」ボタンからモーダルを開き、「対象年月」「相場価格（円/kg）」「出典」を入力して保存します。保存後、未登録バッジが解消されることを確認します。
- **#5 前年同月比の表記（issue #5）の確認**: 相場が登録されている案件（**パターンA: No.123456-a** で確認するのが分かりやすい）では、「前年同月比」ラベルの横に符号付きパーセントが表示されます。未算出の場合は「未算出」と表示され、数値を捏造しないことを確認します。
- 過去経緯パネル: No.123457-a では過去実績がないため空欄（または「該当なし」表示）になることを確認します。
- 自社計画パネル: `plan_id=3`（`spec_id=4`、目標原価率31.0%、計画仕入単価¥680、許容上限¥700）が表示されることを確認します。
- 相場登録＋自社計画確認が完了すると、「次へ」ボタンが活性化し③へ進めます。

### ③ 3ライン算出（`/cases/{caseNo}/lines`）

- 目標／着地／撤退の3ラインが自動算出されることを確認します（暫定算出式 v1）。
- 手修正と修正理由の記録、年間影響額の試算欄を確認します。

### ④ 作戦シート（`/cases/{caseNo}/strategy`）

- 3ライン＋AI生成の交渉ポイント・想定シナリオが表示されることを確認します。数値には必ず引用元（Citation）が添えられ、AIが数値を捏造していないことを確認します。
- **申し送り再利用の確認は、No.123457-a ではなく代表案件②（次節）で行います**（No.123457-a は初回交渉のため申し送りは存在しません）。

### ⑤ 結果記録（`/cases/{caseNo}/result`）

- **#6 所感／申し送りの分離（issue #6）の確認**: 「所感」欄（`staffMemo`、説明文「今回案件の記録（交渉の振り返り）」）と「次回への申し送り」欄（`handoverNote`、説明文「次回案件への判断材料」）が、別セクション・別入力欄として分かれていることを確認します。

## 5. 申し送り再利用の確認手順（代表案件②）

判断継承ループ（BR-10）の実演手順です。**No.123458-a の結果記録 → No.123459-a の情報収集／作戦シートで引用表示される**ことを確認します。

1. `/cases/No.123458-a/result` を開き、⑤結果記録の内容を確認します。
   - 所感: 「輸入牛が為替高・供給ひっ迫で強気。相見積カードで¥1215に収めた」
   - 次回への申し送り: 「次回は別ソースの相見積を早めに用意。小口NGなので数量をまとめて臨む」
   - ステータスは「完了」です。なお現行実装では完了案件でも結果記録フォームは既存内容を復元して再保存でき、参照専用ではありません（`result/page.tsx` は保存済み結果をフォームへ復元します）。誤操作防止はデモ運用側で補足してください。
2. `/cases/No.123459-a/collect` を開き、②情報収集の「過去経緯」パネルに No.123458-a が表示され、上記の申し送り内容が引用されることを確認します（Citation付き。KRE〔知識検索エンジン〕がデータベース由来の過去経緯として即時参照します）。
3. `/cases/No.123459-a/strategy` を開き、④作戦シートに **保存済みの作戦シナリオ**（`sheet_id=7`・シード同梱）が表示され、その内容に No.123458-a の申し送り（別ソースの相見積カード・小口NGへの配慮）が反映されていることを確認します。**この確認は AI 生成を呼ばずに完走できます**（保存済みシートを表示するため）。[🔁再生成] ボタンで AI 再生成を試す場合のみ、Azure OpenAI（`AZURE_OPENAI_*`）の設定が必要です（未設定だと 502 になります）。

> 補足: AI検索の索引（インデックス）・ナレッジグラフ経由の参照は `build_index` 再実行まで反映されない場合があります（[`検収資料.md`](./検収資料.md) 4章参照）。本手順の過去経緯参照はデータベース由来のため、この反映遅延の影響を受けず即時に確認できます。
